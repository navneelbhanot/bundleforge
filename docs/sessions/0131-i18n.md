# Sessions 0131..0136 — server-side i18n

- **M-131** `src/i18n/index.ts` — tiny in-house translator. Loads
  bundled locale JSONs at module init, exposes `t(key, vars?, locale?)`
  with `{var}` interpolation, `resolveLocale(value)` normalizes
  BCP-47/underscored values. Falls back to English on missing keys
  or unknown locales.
- **M-132..M-136** locale JSON files for **en, es, fr, de, it, pt** —
  three string groups (`errors.*`, `checkout.*`, `common.*`) covering
  the most user-facing surfaces today.
- 7 unit tests (English baseline, interpolation, fallback, Spanish
  override, every-locale-loads smoke, BCP-47 normalization).

responses, and webhook acks can speak the merchant's language. M-027's
`shop/update` handler already syncs `Shop.locale`, which downstream
code passes as the third arg to `t()`.
