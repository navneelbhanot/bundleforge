Three guarantees · backed by tests

The differentiators you can’t copy-paste from a feature list.

01

Checkout-parity tested pricing

Most bundle apps compute discounts in two places — the cart widget and the actual checkout. When those drift, the customer sees one price in the cart and a different one at checkout.

BundleForge runs the same pricing function on the server (admin + order processor) and inside Shopify’s Cart Transform Function (live cart + checkout). A test asserts they agree on every commit. Cents-exact, by construction.

02

Inventory audit, enforced at the database

Most apps maintain inventory state in application code. A bug, a misconfigured admin user, or a CSV import can silently rewrite history.

BundleForge writes every adjustment to inventory_audit_log, then enforces immutability at the Postgres trigger level. There is no in-app code path that can rewrite history because the database refuses.

03

A/B testing with real significance

Three competing bundle apps in this category advertise A/B testing. None publish what test of significance is being computed.

BundleForge uses a two-proportion z-test with normal CDF, p-value reported, winner declared only when significant. Property tests run 200 random inputs through each invariant on every commit.

FAQ

Answers we’d want before installing.

Will the cart price ever differ from the checkout price? +

No. The same TypeScript pricing function runs on the server (admin + order webhook) and inside Shopify’s Cart Transform Function (live cart + checkout). A test asserts they agree to the cent on every commit. If they ever disagree the build fails — meaning a drift can’t reach production.

What happens when I hit the Starter plan’s 100-order cap? +

You see a soft warning at 80% of cap and a critical banner at 100%. Bundles already on the storefront keep working; new bundle orders bypass discount logic until you upgrade or the calendar month ticks over. Nothing is silently throttled. The Growth plan ($12/mo) removes the cap entirely.

Can I cancel? Will I be refunded? +

Cancel any time from the Billing tab. Monthly plans stop billing at the next cycle. Annual plans are refunded pro-rata for the unused months — see ToS §3. Your bundle data stays exportable for 30 days after uninstall.

Where does my bundle data live? Can I export it? +

Postgres in eu-west (Railway), encrypted at rest. A single GET endpoint returns every bundle, pricing rule, and audit-log row for your shop as JSON — exportable on demand, not hostage to a CSV button. GDPR data-export and shop-redact webhooks are wired and tested.

Does BundleForge work with Shopify Plus? +

Yes. The Cart Transform Function approach is the modern Plus-friendly path — it doesn’t edit checkout.liquid and doesn’t fight Functions-based discount apps. Multi-currency and B2B markups are first-class on Pro and Enterprise.

POS, Hydrogen, headless — supported? +

POS single-location on Growth, multi-location on Pro. Headless storefronts hit the public Storefront API endpoint with a token; bundle data and current pricing come back as JSON. Hydrogen example components live under extensions/.

How do you guarantee inventory consistency? +

Every adjustment writes to inventory_audit_log inside a single Postgres transaction. The table has REVOKE UPDATE, DELETE applied at the database level — even a future bug or a wayward admin can’t silently rewrite history. The order webhook reconciles bundle-order components to Shopify’s authoritative inventory at fulfilment time.

Is BundleForge GDPR-compliant? +

The three Shopify-mandated GDPR webhooks (customers/data_request, customers/redact, shop/redact) are implemented, tested, and idempotent. We don’t collect customer PII beyond what’s in the Shopify order payload, and we don’t resell or share data with third parties.

Trust, but verify

Every claim above is reproducible.

Each guarantee on this page maps to a test in the BundleForge repo. git clone, run npm test, and the assertions either pass or they don’t. Nothing here relies on you taking our word for it.

All vitest + Playwright suites green on the working branch. Pricing-parity, atomic inventory, and significance calculator each have dedicated test files.
